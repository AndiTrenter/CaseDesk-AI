The user wants to build a self-hosted, privacy-focused, modular web application named CaseDesk AI. This application is intended for private and business use for managing documents, emails, calendars, and cases with AI assistance.

**PRODUCT REQUIREMENTS:**
- **100% Self-Hosted:** Must run entirely independently via Docker Compose, with no runtime dependencies on the Emergent platform.
- **Setup Wizard:** An initial setup process for creating the admin user, configuring email accounts, and setting up AI providers (local or external).
- **Multi-Tenancy & User Management:** Support for multiple users with strictly separated data. Admins should be able to invite new users via email links.
- **Document Processing:** Upload various formats (PDF, DOCX, images), with OCR for scanned documents, metadata extraction, full-text and semantic search, and association with cases. - **Intelligent Renaming:** Documents should be automatically renamed to the format: .
- **Email Processing:** Fetch emails and attachments via IMAP, analyze content, identify tasks/deadlines, and mark emails as read after processing. SMTP settings should be configurable for sending emails.
- **Configurable & Multilingual AI:**
    - Allow users to choose between a local LLM (like Ollama) or an external one (like OpenAI).
    - The AI must always respond in the language set in the user's preferences.
    - The AI should assist by answering questions, summarizing, drafting replies, and finding missing documents, but MUST NOT invent facts.
    - The AI should have knowledge of all documents belonging to a user to provide context-aware assistance and suggest cross-references.
- **Proactive AI Assistant:** The AI should proactively prepare information for the user, such as a daily briefing, suggesting relevant documents for a new case, and providing analysis within a case.
- **AI-Powered Response Generation:** The AI assistant must be able to take a case, generate a complete response letter (in selectable formats like PDF or DOCX), automatically find and attach all necessary documents from that case, and provide the complete package for download or sending via email.
- **Core Features:** Calendar, task management, and an audit trail to track actions within a case.
- **No Branding:** The application must not contain any Made with Emergent watermarks or branding.

**what currently exists?**
A comprehensive full-stack application with a FastAPI backend, React frontend, and PostgreSQL database, all containerized with Docker.

Key implemented features:
- **Core:** Setup wizard, JWT authentication, multi-user support with an email invitation system, and a light/dark theme toggle.
- **Document Management:** Document upload with OCR, intelligent renaming, and semantic search. Users can perform multi-select on documents to assign them to a case, and also upload/link documents from within a case view.
- **Case Management:** A case detail page with tabs for documents, correspondence, and history.
- **AI Integration:** An AI abstraction layer supports both local (Ollama) and external (OpenAI) models. The AI assistant has access to all of the user's documents for context-aware chat.
- **Proactive AI:** A Daily Briefing on the dashboard, document suggestions during case creation, and a proactive analysis tab within cases.
- **Response Generation:** Backend and frontend infrastructure for generating a response package for a case, including a drafted letter and attachments.
- **Settings:** UI for managing AI providers, IMAP/SMTP email accounts, and user administration (invitations).
